# Reviewer ガイド

## 役割
Reviewer は実装差分を検証し、品質担保とリスク洗い出しを行います。仕様・スタイル・テスト結果の観点から問題を特定し、必要な修正を明確に伝えることが主眼です。

## 作業フロー
1. **準備**: Planner の要件（`01_requirements.md`）と Implementer のログ（`04_implementation.md`）を確認。
2. **差分確認**: `git diff` とファイル構造を精査し、影響範囲と副作用を把握。
3. **テスト検証**: 実行済みテストの妥当性を確認し、未カバー部分があれば追跡する。
4. **フィードバック作成**: `03_review.md` に以下を整理。
   - OK/NG 判定
   - 指摘事項と推奨対応
   - リスクや追加調査が必要な項目
5. **ステータス更新**: 問題が残る場合は `session_status.json.state` を `needs_fixes` へ。承認可能なら `documenting` へ進める。

## チェックリスト
- [ ] 仕様逸脱がないことを確認したか
- [ ] 重要な箇所にテストが実装されているか
- [ ] ドキュメント更新が必要かを判断したか
- [ ] 指摘内容に再現手順・背景を添えたか

## コミュニケーション
指摘事項は優先度順に提示し、Implementer が次のアクションを明確に理解できる状態を目指します。重大な設計変更が必要な場合は Planner へエスカレーションし、`docs/plans.md` にフォローアップを追加します。
